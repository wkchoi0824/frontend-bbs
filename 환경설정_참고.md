프로젝트 시작 체크리스트 (헷갈릴 때 보는 요약)

## 프로젝트마다 새로 해야 하는 것
1. 패키지 설치
- 프론트: `npm install` / `pnpm install`
  > 둘 중 하나만 사용. `package-lock.json`이면 npm, `pnpm-lock.yaml`이면 pnpm 사용

- 백엔드: `mvn install` / `gradle build`
  > 둘 중 하나만 사용. `pom.xml`이면 Maven, `build.gradle`/`build.gradle.kts`이면 Gradle

2. 환경설정 파일
- `.env`, `application.yml`, `application.properties`

3. 프로젝트별 설정
- 라우팅, CORS, DB 연결, 빌드 설정

4. 폴더 구조/코드 작성


## 한 번만 하면 되는 것(PC 전역)
1. VSCode Extensions 설치

2. 개발 도구 설치
- Node.js, JDK, Git, Docker 등

3. 전역 도구 설정
- Git 사용자명/이메일, npm/pnpm 전역 설정 등

짧은 기억 공식
- "프로젝트 안에 있는 것"은 매번 새로
- "PC에 설치하는 것"은 한 번만

# 추가 참고: Gradle/Spring Initializr와 프론트 패키지 매니저
- `package-lock.json`은 npm 실행 시 생성되는 프론트 잠금파일이며 Gradle과 무관
- pnpm을 쓰려면 `package-lock.json`을 제거하고 `pnpm install` 실행
- 생성되는 잠금파일 기준: npm → `package-lock.json`, pnpm → `pnpm-lock.yaml`


# VS Code + Spring Initializr 상세 가이드(Gradle 기준)

1. 명령 팔레트 열기
Windows: `Ctrl + Shift + P`
Mac: `Cmd + Shift + P`

2. 명령어 입력
`Spring Initializr: Create a Gradle Project` 입력 후 Enter

3. 확장 설치 확인
확장 보기: `Ctrl + Shift + X` 또는 `Cmd + Shift + X`
필수 확장: `Extension Pack for Java`, `Spring Boot Extension Pack`

4. 스프링 부트 버전 선택
최신 정식 버전(예: `3.2.x`). Snapshot, M1 표시는 피하기

5. 언어 선택
`Java`

6. Group ID 입력
도메인을 거꾸로. 예: `com.mentor`

7. Artifact ID 입력
프로젝트 이름. 예: `blog-backend`

8. Packaging 선택
`Jar`

9. Java 버전 선택
수업 기준 `17`

10. 의존성 추가
`Spring Web`, `Lombok`, `Spring Data JPA`, `MariaDB Driver`, `Spring Security`

    # 추가 의존성 3종 (쓰임새 + 예시)
    1. Validation
    - 입력값 검증용. 잘못된 요청을 서버에서 차단
    - 예: 회원가입/게시글 작성 시 이메일 형식, 비밀번호 길이, 제목 최소 글자 수 검증
    - 간단 예시
    ```java
    public class SignupRequest {
    @NotBlank
    private String username;

    @Email
    private String email;

    @Size(min = 8)
    private String password;
    }

    @PostMapping("/signup")
    public ResponseEntity<?> signup(@Valid @RequestBody SignupRequest req) { ... }
    ```
    2. Spring Boot DevTools
    - 개발 중 자동 재시작/리로드로 작업 속도 향상
    - 예: 컨트롤러 수정 후 저장만 해도 서버가 다시 올라옴
    3. Spring Boot Actuator
    - 운영/모니터링용 상태 확인 엔드포인트 제공
    - 예: `/actuator/health`로 서버 상태 확인

11. 저장 폴더 선택 후 프로젝트 생성

12. 프로젝트 구조 핵심만 확인
`src/main/java`: Java 코드 위치
`src/main/resources/application.yml`: 환경설정 위치
`build.gradle`: 의존성 명세서

13. 첫 실행 테스트
Spring Boot Dashboard에서 프로젝트 옆의 Play 버튼 클릭
터미널 로그에서 구동 여부 확인
DB 설정 전에는 연결 오류가 날 수 있음(정상적인 초기 상태)

## Gradle/Spring Initializr 설치 및 준비 체크
- JDK 설치 (수업 기준: JDK 17)
- `JAVA_HOME` 환경 변수 설정
- Spring Initializr에서 프로젝트 생성 시 선택
- Gradle 또는 Maven 중 하나만 선택
- 생성 후 확인 파일
  - Gradle: `build.gradle` 또는 `build.gradle.kts`
  - Maven: `pom.xml`

## Gradle 기준 설치/준비 순서(처음부터 끝까지)
1. JDK 17 설치
2. `JAVA_HOME` 설정 및 `java -version` 확인
3. IDE/에디터 설치 (VSCode 또는 IntelliJ)
4. Spring Initializr에서 프로젝트 생성
- Project: Gradle
- Language: Java
- Java: 17
5. 프로젝트 열기
6. Gradle 동기화 (IDE 안내에 따라)
7. 첫 빌드 확인: `./gradlew build` 또는 `gradlew build`
